01

Project Overview

Jyorah Business Solutions is a premium accounting, VAT, bookkeeping, payroll and business support firm based in Dubai, UAE. This website (jyorah.com) is the company's primary digital presence — a single-page marketing site designed to attract and convert UAE business clients.

The site is built as a single static HTML file (index.html) with all styles and scripts inline. It is hosted on Vercel, which runs two lightweight serverless functions in the /api/ directory: one serves runtime credentials (/api/env-config) and another proxies the n8n webhook to bypass browser CORS restrictions (/api/n8n).

Lead capture is handled client-side: the consultation form POSTs directly to the Airtable REST API (saves the lead) and sends a notification email via EmailJS (internal alert to info@jyorah.com). Simultaneously, it fires a webhook to n8n which automatically sends a personalised HTML reply email back to the customer.

Local Development Setup

  1. Clone the repository
    git clone https://github.com/aivarya/jyorah.git
    Then cd jyorah
  2. Install dependencies
    npm install
    This installs Puppeteer (used only for screenshots). No other packages needed.
  3. Create your .env file
    Copy the template: cp .env.example .env
    Fill in all 7 credentials (see Section 5). The N8N_WEBHOOK_URL can be left blank locally if not testing auto-reply.
  4. Start the dev server
    node serve.mjs
    Server starts at http://localhost:3000 — mirrors the Vercel serverless behaviour locally.
  5. Open in browser
    Navigate to http://localhost:3000
    The site loads with credentials injected via /env-config.js
  6. (Optional) Take a screenshot
    node screenshot.mjs http://localhost:3000 my-label
    Saved to temporary screenshots/screenshot-N-my-label.png
The server must be running at localhost:3000 (not a file:/// URL) for the /env-config.js endpoint to work and for credentials to be loaded correctly. On Vercel, this is handled automatically by api/env-config.js.

How Credentials Are Loaded

Since the site is a static HTML file with no server-side rendering, credentials are loaded at runtime via a dedicated endpoint. The same pattern is used both locally and in production — only the handler differs.

Local Development (serve.mjs)

  1. serve.mjs reads .env from disk when the server starts
  2. When the browser requests /env-config.js, serve.mjs responds with JavaScript that sets window._JYORAH
  3. index.html loads <script src="/env-config.js"></script> in <head>
  4. The form submission function reads credentials from window._JYORAH at submit time

Production (Vercel — api/env-config.js)

  1. vercel.json rewrites any request to /env-config.js → /api/env-config
  2. api/env-config.js reads environment variables from process.env (set in Vercel dashboard)
  3. Returns the same window._JYORAH = {...} JavaScript response
  4. The browser receives it identically — no change needed in index.html

Response from /env-config.js

window._JYORAH = {
  airtableBaseId:    "appfO9KcyKCqvigDU",
  airtableTableName: "Consultations",
  airtableToken:     "patXXXXXXXXXXXXXX...",
  emailjsServiceId:  "service_63wuxhw",
  emailjsTemplateId: "template_tyjm9qu",
  emailjsPublicKey:  "72-fPQVl8Gl4YC3_J",
  n8nWebhookUrl:     "/api/n8n"
};
Security note: These values ARE visible in the browser (DevTools → Network → env-config.js). The .env / Vercel env var pattern prevents credentials from being committed to Git — it does not hide them from someone viewing DevTools. The Airtable token is scoped to data.records:write on this base only, limiting exposure.

Airtable Integration

Each form submission creates a new record in the Consultations table of the Jyorah Leads Airtable base.

API Call

POST https://api.airtable.com/v0/{airtableBaseId}/{airtableTableName}

Headers:
  Authorization: Bearer {airtableToken}
  Content-Type:  application/json

Body:
{
  "fields": {
    "Timestamp": "2026-05-21T10:30:00.000Z",
    "Full Name": "Ahmed Al-Rashid",
    "Email":     "ahmed@example.com",
    "Mobile":    "+971 50 123 4567",
    "Service":   "VAT Filing & Compliance",
    "Message":   "Need help with Q1 VAT return"
  }
}

Access Token Permissions

The Personal Access Token must have:

  • Scope: data.records:write
  • Base access: this base only (not all bases)

This limits exposure — a leaked token can only add records to this specific base, not read, delete, or access other bases.

Accessing Leads

Log into airtable.com with the account that owns the Jyorah Leads base. All form submissions appear in the Consultations table in real time.

EmailJS Integration

EmailJS sends a notification email to info@jyorah.com each time the consultation form is submitted. This is the internal alert to the Jyorah team — it is separate from the customer auto-reply which is handled by n8n. n8n Auto-Reply

Live n8n.sreyx.com

n8n is a self-hosted workflow automation tool running at n8n.sreyx.com. When a customer submits the consultation form, n8n automatically sends them a personalised, branded HTML reply email within seconds.

The website does not depend on n8n. Leads are saved to Airtable and the internal notification goes via EmailJS regardless of whether n8n is online. n8n is only responsible for the customer-facing auto-reply.

Managing the Workflow

  1. Log in to n8n.sreyx.com
  2. Go to Workflows → open Jyorah → Customer Auto Reply
  3. To view past executions: click Executions (top bar) — each run is logged with input/output per node
  4. To update SMTP settings: go to Credentials → SMTP account 2
  5. To update the AI prompt or email template: open the AI Agent or Parse Email JSON nodes

Setting Environment Variables on Vercel

  1. Log in to vercel.com → open the jyorah project
  2. Go to Settings → Environment Variables
  3. Add all 7 variables (same names as the .env file — see Section 5). Set scope to Production (and Preview if needed).
  4. After adding/updating variables, redeploy the project — Vercel only injects env vars at deploy time, not live.
Changing an environment variable on Vercel does not take effect until you trigger a new deployment. Go to the Deployments tab → click the latest deployment → Redeploy (or push a commit to main).

Issue Lookup Table

Symptom Platform Where Exactly What to Look For / Fix
Customer auto-reply email not received n8n n8n.sreyx.com → Workflows → "Jyorah → Customer Auto Reply" → Executions Look for red nodes. Click the red node to see the error message. Common causes: SMTP auth failed, wrong password, port blocked. Go to Credentials → SMTP account 2 → click Test.
n8n email shows green but still not received n8n Same execution → Send Reply Email node output continueOnFail may be swallowing an error. Set it to false temporarily to expose the real error. Also check spam/junk folder.
Lead not saved in Airtable Browser + Airtable F12 → Network tab → filter by "airtable" → check the POST request status 401 = invalid API token. 422 = field name mismatch (check exact column names in base). 429 = rate limited. Verify AIRTABLE_API_TOKEN in Vercel env vars.
Internal notification not received (info@jyorah.com) EmailJS emailjs.com → Dashboard → Usage / Statistics Check if 200/month free tier limit is reached. Check template is active and not paused. Check spam folder.
Website not loading or showing 500 error Vercel vercel.com → jyorah project → Deployments tab Check if latest deployment is green. Click a failed deployment to see build logs. A red deployment = site is serving the last good version.
Form submits but nothing happens / no success message Browser F12 → Console tab (look for errors) + Network tab (look for failed requests) Most likely /env-config.js failed to load (check Network → env-config.js response). Could also be an Airtable 4xx error stopping execution.
n8n webhook not being triggered at all Vercel vercel.com → jyorah project → Functions tab → /api/n8n → Logs Check if the Vercel function is receiving the POST. If no logs appear, the form JS may not be calling it. Verify N8N_WEBHOOK_URL env var is set in Vercel settings.
SMTP error in n8n execution n8n n8n.sreyx.com → Credentials → SMTP account 2 → Test Common fixes: wrong host/port, app password needed (if using Gmail/Outlook), Secure (SSL) toggle wrong. Port 587 = STARTTLS (Secure OFF), Port 465 = SSL (Secure ON).
All outgoing emails landing in spam DNS Check jyorah.com DNS records for SPF, DKIM, DMARC via mxtoolbox.com Missing SPF record is the most common cause. Add SPF record for your mail provider. DKIM and DMARC improve deliverability further.
Environment variable change not taking effect Vercel vercel.com → jyorah → Settings → Environment Variables Vercel only applies env var changes on redeploy. After editing a variable, go to Deployments → Redeploy, or push a new commit to trigger a fresh deployment.
AI Agent in n8n generating wrong / poor replies n8n n8n.sreyx.com → Workflow → AI Agent node → System Message Edit the system prompt to adjust tone, length, or format. You can also pin test data to the Webhook node and run the workflow manually to preview output without a real form submission.

General Debugging Steps

  1. Identify which step failed — is it saving to Airtable, sending the internal EmailJS alert, or the n8n auto-reply? Each is independent.
  2. Check the right platform — use the table above to go directly to the relevant dashboard.
  3. Look at logs — n8n Executions, Vercel Function logs, and browser DevTools Network tab are the three key places.
  4. Test credentials in isolation — n8n has a credential test button. Vercel env vars can be verified in the Functions → Logs output.
  5. Check spam/junk — both the internal EmailJS alert and the n8n auto-reply can end up in spam if SPF/DKIM records are not set up for jyorah.com.
