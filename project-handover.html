Project Overview

Finora Business Solutions is a premium accounting, VAT, bookkeeping, payroll and business support firm based in Dubai, UAE. This website (finorame.com) is the company's primary digital presence — a single-page marketing site designed to attract and convert UAE business clients.

The site is built as a single static HTML file (index.html) with all styles and scripts inline. A lightweight Node.js HTTP server (serve.mjs) serves the file locally and provides one special route (/env-config.js) that injects API credentials from a .env file into the browser at runtime — keeping secrets out of source control.

Lead capture is handled entirely client-side: the consultation form POSTs directly to the Airtable REST API and sends a notification email via EmailJS. There is no application backend. All leads are stored in Airtable for later processing by n8n automation.

Local Development Setup

  1. Clone the repository
    git clone https://github.com/aivarya/finorame.git
    Then cd finorame
  2. Install dependencies
    npm install
    This installs Puppeteer (used only for screenshots). No other packages needed.
  3. Create your .env file
    Copy the template: cp .env.example .env
    Fill in your real Airtable and EmailJS credentials (see Section 5).
  4. Start the dev server
    node serve.mjs
    Server starts at http://localhost:3000
  5. Open in browser
    Navigate to http://localhost:3000
    The site loads with credentials injected via /env-config.js
  6. (Optional) Take a screenshot
    node screenshot.mjs http://localhost:3000 my-label
    Saved to temporary screenshots/screenshot-N-my-label.png
The server must be running at localhost:3000 (not a file:/// URL) for the /env-config.js endpoint to work and for credentials to be loaded correctly.

How Credentials Are Loaded

Since the site is a static HTML file with no server-side rendering, credentials can't be embedded at build time. The pattern used here is:

  1. serve.mjs reads .env from disk when the server starts
  2. When the browser requests /env-config.js, the server responds with a JavaScript file that sets window._FINORA
  3. index.html loads <script src="/env-config.js"></script> in <head>
  4. The form submission function reads credentials from window._FINORA at submit time

Response from /env-config.js

window._FINORA = {
  airtableBaseId:    "appfO9KcyKCqvigDU",
  airtableTableName: "Consultations",
  airtableToken:     "patXXXXXXXXXXXXXX...",
  emailjsServiceId:  "service_63wuxhw",
  emailjsTemplateId: "template_tyjm9qu",
  emailjsPublicKey:  "72-fPQVl8Gl4YC3_J"
};
Security note: These values ARE visible in the browser (DevTools → Network → env-config.js). The .env pattern prevents credentials from being committed to Git — it does not hide them from someone viewing page source. The Airtable token is scoped to data.records:write on this base only, limiting exposure to someone being able to add records (not read or delete them).
For production hosting (Render, Railway, etc.), set the six variables as environment variables in the hosting dashboard. The serve.mjs server reads them via the same process.env mechanism.

Submission Flow

User opens modal (optionally with pre-selected service) │ ▼ Fills form → clicks "Submit Request" │ ▼ submitConsult() fires │ ├──────────────────────────────────┐ ▼ ▼ POST → Airtable API emailjs.send() (lead saved to spreadsheet) (notification to info@finorame.com) │ │ └──────────────┬───────────────────┘ ▼ Both wrapped in try/catch (one failing does not block the other) │ ▼ Success state shown "Request Received" │ ▼ User clicks Close → modal dismisses

Modal Close Behaviour

The modal closes only via the ✕ button (top-right) or the "Close" button on the success screen. Clicking the backdrop or pressing Escape does not close it — this prevents accidental loss of partially filled form data.

Airtable Integration

Each form submission creates a new record in the Consultations table of the Finora Leads Airtable base.

API Call

POST https://api.airtable.com/v0/{airtableBaseId}/{airtableTableName}

Headers:
  Authorization: Bearer {airtableToken}
  Content-Type:  application/json

Body:
{
  "fields": {
    "Timestamp": "2026-05-21T10:30:00.000Z",
    "Full Name": "Ahmed Al-Rashid",
    "Email":     "ahmed@example.com",
    "Mobile":    "+971 50 123 4567",
    "Service":   "VAT Filing & Compliance",
    "Message":   "Need help with Q1 VAT return"
  }
}

Access Token Permissions

The Personal Access Token must have:

  • Scope: data.records:write
  • Base access: this base only (not all bases)

This limits exposure — a leaked token can only add records to this specific base, not read, delete, or access other bases.

Accessing Leads

Log into airtable.com with the account that owns the Finora Leads base. All form submissions appear in the Consultations table in real time.

n8n Automation (Planned)

Planned — Not Yet Live

n8n is a self-hosted workflow automation tool. The client runs an instance locally. The planned workflow will:

  1. Monitor the Airtable Consultations table for new records (polling or webhook)
  2. Compose a personalised reply email to the lead using their submitted details
  3. Send the reply via Outlook using Microsoft account jyothi@finorame.com
Important: The website form does not depend on n8n. Leads are captured in Airtable and the internal notification is sent via EmailJS regardless of whether n8n is online. n8n is only responsible for the automated reply back to the lead.

Deployment Notes

The website currently runs locally via node serve.mjs. To publish it publicly, a Node.js hosting environment is required so that serve.mjs can run and serve the /env-config.js endpoint.

Recommended: Render (Free Tier)

  1. Connect your GitHub repo (aivarya/finorame) to render.com
  2. Create a Web Service — Runtime: Node · Build command: npm install · Start command: node serve.mjs
  3. Add all 6 environment variables in Render's dashboard (Environment tab) — no .env file needed on the server
  4. Deploy — Render provides a public HTTPS URL. Point your domain finorame.com to it via a CNAME record

Why Not Static Hosting?

GitHub Pages, Netlify, and Vercel serve purely static files. They cannot run serve.mjs or respond to the /env-config.js route. Deploying to these platforms as-is will cause the form to silently fail (credentials never loaded). A Node.js host is required, OR the /env-config.js pattern must be replaced with build-time env injection (e.g. Vite's import.meta.env).

Custom Domain (finorame.com)

StepAction
1Add finorame.com as a custom domain in Render dashboard
2Render provides a CNAME target (e.g. finorame.onrender.com)
3In your domain registrar (GoDaddy / Namecheap / Cloudflare etc.) add a CNAME record: finorame.com → finorame.onrender.com
4Render provisions an SSL certificate automatically (Let's Encrypt)
